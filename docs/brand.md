# gameengine.js Brand Kit

## Core identity
- **Tagline:** Cursor for gaming – chat commands become playable worlds.
- **Values:** approachable, multiplayer-by-default mindset, builder energy, radical transparency.

## Logo + wordmark
- **Wordmark:** lowercase `gameengine.js` in Space Grotesk SemiBold, tracking -2.
- **Icon:** stacked `ge` monogram inside a rounded 8px square.
- **Usage:** icon only for avatars, full wordmark for headers.

## Colors
| Name | Hex | Usage |
| --- | --- | --- |
| Midnight Grid | `#05060A` | App background, hero gradients |
| Neon Pulse | `#7C3AED` | Primary CTA, highlights |
| Citrus Spark | `#FACC15` | Accent, notification badges |
| Signal Teal | `#2DD4BF` | Success states, interactive affordances |
| Soft Slate | `#94A3B8` | Supporting text, borders |

## Typography
- **Display / Headings:** Space Grotesk (600/700)
- **Body:** Inter (400/500)
- **Monospace:** IBM Plex Mono (for code/prompt snippets)

## Tone & voice
- Speak in second person (“Describe your scene… we’ll build it”).
- Keep copy playful but direct—“vibe coding” should feel powerful, not silly.
- Always highlight “2D today, 3D tomorrow”.

## Landing page sections (draft copy)
1. **Hero:**
   > “Ship a playable game in minutes. Describe your scene in chat, export to web/desktop/mobile.

   CTA buttons: `Launch editor` (primary) / `See templates` (secondary).

2. **How it works:** 3 steps – Describe → Preview → Ship.
3. **Included batteries:** built-in asset packs, donation-supported hosting, community remixing.
4. **Roadmap highlight:** show Phase 0–3 timeline.
5. **Open source & donations:** MIT license, OpenCollective + Ko-fi buttons.

## Imagery guidelines
- Use neon gradients on dark surfaces, screenshots of chat + canvas side-by-side.
- Showcase template thumbnails as trading-card grid.
- Prefer abstract cityscapes / holographic controllers over literal console photos.
